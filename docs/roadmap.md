# Roadmap

## Current MVP

- public GitHub repository campaigns
- sponsor-funded SOL escrow at campaign creation
- deadline-based campaign lifecycle
- GitHub-authenticated user flows
- allocation pipeline from GitHub activity
- AI-assisted allocation with deterministic fallback
- finalize preview and on-chain finalization
- contributor claim flow
- sponsor refund flow after claim deadline
- campaign listing and campaign details UI
- my campaigns / my claims views
- final allocation display in UI

## Next

- durable multi-batch finalization orchestration
- backend-paid claim fee flow
- close-unfinalizable workflow for operator/backend use
- stronger campaign-to-chain reconciliation and recovery flows
- better contributor metrics beyond the current MVP approximation
- clearer allocation explanations in UI
- improved campaign history and audit visibility
- stronger operational monitoring and service-wallet health checks

## Future

- GitHub App integration
- richer contributor scoring inputs, including reviews and issue activity
- dispute and review mechanics for finalized allocations
- multisig / stronger authority separation
- SPL token support beyond SOL
- contributor notifications
- anti-spam and bot filtering
- reputation-aware reward models
- multi-sponsor campaigns
- production-grade analytics and observability
